# AndroidLab 🧪

A personal Android development playground for learning and experimenting with Android technologies.
Each topic lives in its own feature module with a working implementation, unit tests, and dedicated documentation.

## Philosophy

- **Learn by building** — every module contains a real, working implementation
- **Tests first** — each topic starts with a test scaffold generated by AI, implementation is manual
- **AI for docs** — code is written by hand, descriptions and explanations are AI-generated
- **Visual** — each feature module has a dedicated screen showing the concept in action

## Project Structure
```
AndroidLab/
├── app/                    # Main activity, navigation, Koin setup
├── core/                   # Shared logic: models, utilities, base classes
├── core-test/              # Shared test helpers, fakes, and utilities
└── feature-*/              # Individual learning modules (one topic each)
```

## Tech Stack

| Layer | Technology |
|---|---|
| Language | Kotlin |
| UI | Jetpack Compose + Material3 |
| Navigation | Navigation Compose |
| DI | Koin |
| Async | Coroutines + Flow |
| Network | Ktor, Retrofit |
| Database | Room |
| Serialization | Kotlinx Serialization |
| Testing | JUnit5 + MockK + Turbine |
| CI | GitHub Actions |

## Learning Workflow

Each new topic follows the same pattern:

1. **Define the topic** — pick a concept, library, or pattern to explore
2. **Generate scaffold** — ask AI (Gemini in Android Studio) for a task description, boilerplate code, and test cases
3. **Implement manually** — write the actual implementation guided by tests
4. **Document** — ask AI to generate the explanation and description for the module README
5. **Commit** — the module becomes a permanent reference
